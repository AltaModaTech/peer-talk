Peer Discovery

Various schemes are used to find other peers in the network.

To get the discovered peers use the Swarm.KnownPeers property.

foreach (var peer in swarm.KnownPeers)
{
   Console.WriteLine(peer.Id);
}

Schemes

Multicast DNS

MdnsNext uses RFC 6762 - Multicast DNS to locate peers on the local area network with zero configuration. Local area network peers are very useful to peer-to-peer protocols, because of their low latency links.

The mDNS-discovery specification describes how IPFS uses mDNS to discover other peers; private networks are also supported.

Bootstrap

Bootstrap uses a pre-configured list of addresses of highly stable (and somewhat trusted) peers to find the rest of the network. These is sometimes referred to as seed nodes.

Randon walk

The Ipfs.Engine makes random DHT (Distributed Hash Table) queries in order to discover a large number of peers quickly. This causes the DHT to converge much faster, at the expense of a small load at the very beginning.
